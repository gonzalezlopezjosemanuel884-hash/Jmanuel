1. HTML: La Estructura de tu Web

HTML (HyperText Markup Language) es el lenguaje que da forma al contenido de tu página. Define la jerarquía de títulos, párrafos, listas, enlaces, imágenes, tablas, formularios y mucho más. Piensa en HTML como el esqueleto de tu sitio.

1.1. La Estructura Básica

Toda página HTML debe tener esta estructura mínima:

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Mi primera web</title>
</head>
<body>
  <h1>¡Hola mundo!</h1>
  <p>Este es mi primer párrafo.</p>
</body>
</html>

Explicación: <!DOCTYPE html> declara el tipo de documento. El elemento <html> es la raíz. <head> contiene metadatos como el título y la codificación. <body> alberga todo el contenido visible.

1.2. Etiquetas Esenciales

📝 Encabezados y párrafos

Los encabezados van de <h1> a <h6>. <h1> es el más importante (solo uno por página). Los párrafos se crean con <p>.

🔗 Enlaces

El elemento <a> crea enlaces. El atributo href define el destino. Ejemplo: <a href="https://google.com">Ir a Google</a>.

🖼️ Imágenes

La etiqueta <img> inserta imágenes. Requiere el atributo src (ruta de la imagen) y alt (texto alternativo).

📋 Listas

<ul> para listas desordenadas (viñetas), <ol> para listas ordenadas (numeradas). Cada ítem se define con <li>.

1.3. Tablas y Formularios

Las tablas se construyen con <table>, <tr> (filas), <th> (encabezados) y <td> (celdas). Los formularios (<form>) recogen datos del usuario con campos como <input>, <textarea> y <select>.

<form action="/enviar" method="POST">
  <label for="nombre">Nombre:</label>
  <input type="text" id="nombre" name="nombre">
  <input type="submit" value="Enviar">
</form>

💡 Consejo: HTML semántico significa usar etiquetas con significado (como <header>, <nav>, <section>, <article>, <footer>) en lugar de solo <div>. Esto mejora la accesibilidad y el SEO.

2. CSS: Dale Estilo y Personalidad

CSS (Cascading Style Sheets) es el lenguaje que define la presentación visual. Con CSS puedes cambiar colores, fuentes, tamaños, márgenes, posicionamiento, animaciones y diseño responsivo. Es el maquillaje que hace atractiva tu web.

2.1. Selectores y Propiedades

Los selectores apuntan a elementos HTML. Los más comunes son:

  • Selector de etiqueta: h1 { color: red; }
  • Selector de clase: .clase { font-size: 20px; } (se aplica con class="clase")
  • Selector de ID: #id { background: blue; } (se aplica con id="id")
/* Ejemplo de CSS */
/* Estilo para todos los párrafos */
p {
  color: #333;
  font-family: Arial, sans-serif;
  line-height: 1.6;
}

/* Clase para destacar */
.destacado {
  background-color: #facc15;
  padding: 10px;
  border-radius: 8px;
}

2.2. Modelo de Caja (Box Model)

Todo elemento en CSS es una caja compuesta por: content (contenido), padding (relleno interior), border (borde) y margin (margen exterior). Entender esto es clave para diseñar layouts.

.caja {
  width: 300px;
  padding: 20px;
  border: 2px solid #333;
  margin: 15px;
  background: #f8fafc;
}

2.3. Flexbox y Grid (Layouts Modernos)

Flexbox es ideal para diseños unidimensionales (filas o columnas). Grid es perfecto para diseños bidimensionales (filas y columnas). Ambos son herramientas imprescindibles.

.flex-container {
  display: flex;
  justify-content: center;
  align-items: center;
  gap: 20px;
}

.grid-container {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  gap: 15px;
}

2.4. Diseño Responsivo (Media Queries)

Para que tu web se vea bien en móviles, usa media queries. Ajustan los estilos según el tamaño de la pantalla.

/* Para pantallas menores a 768px */
@media (max-width: 768px) {
  .grid-container {
    grid-template-columns: 1fr;
  }
  h1 { font-size: 1.8rem; }
}

3. JavaScript: La Interactividad

JavaScript es el lenguaje que da vida a tu web. Permite responder a eventos (clics, movimientos de ratón, teclado), manipular el contenido dinámicamente, hacer animaciones, validar formularios, consumir APIs y mucho más.

3.1. Sintaxis Básica

JavaScript se escribe en el archivo .js o dentro de <script> en el HTML.

// Variables
let nombre = "José Manuel";
const edad = 17;
var ciudad = "Cuba"; // (obsoleto, mejor usar let/const)

// Funciones
function saludar() {
  alert("¡Hola " + nombre + "!");
}

// Condicionales
if (edad >= 18) {
  console.log("Eres mayor de edad");
} else {
  console.log("Eres menor de edad");
}

// Bucles
for (let i = 0; i < 5; i++) {
  console.log("Iteración " + i);
}

3.2. Manipulación del DOM

El DOM (Document Object Model) es la representación en JavaScript de tu página. Puedes seleccionar elementos y modificarlos.

// Seleccionar un elemento
let titulo = document.querySelector('h1');
titulo.textContent = "Nuevo título";
titulo.style.color = "blue";

// Añadir un evento
let boton = document.querySelector('#miBoton');
boton.addEventListener('click', function() {
  alert("¡Clickeado!");
});

3.3. Eventos Comunes

  • click – al hacer clic en un elemento.
  • mouseover – al pasar el ratón por encima.
  • keydown – al presionar una tecla.
  • submit – al enviar un formulario.
  • load – cuando la página termina de cargar.
// Formulario que previene el envío y muestra los datos
document.querySelector('form').addEventListener('submit', function(e) {
  e.preventDefault(); // Evita que se recargue la página
  let nombre = document.querySelector('#nombre').value;
  alert("Hola " + nombre + ", formulario enviado");
});

🔥 Pro tip: Usa console.log() para depurar tu código. Abre las herramientas de desarrollador (F12) y ve la consola para ver mensajes y errores.

3.4. Almacenamiento Local (localStorage)

Permite guardar datos en el navegador del usuario de forma persistente.

// Guardar datos
localStorage.setItem('nombre', 'José Manuel');

// Recuperar datos
let nombreGuardado = localStorage.getItem('nombre');
console.log(nombreGuardado);

// Eliminar dato
localStorage.removeItem('nombre');

4. Buenas Prácticas y Consejos de Profesional

  • Escribe código limpio y comentado: Los comentarios ayudan a ti y a otros a entender tu código.
  • Usa nombres descriptivos: En lugar de a o b, usa nombreUsuario o precioTotal.
  • Optimiza imágenes: Comprime imágenes para que tu web cargue rápido.
  • Prueba en varios navegadores: Asegúrate de que tu web funcione en Chrome, Firefox, Edge y Safari.
  • Mantén separados HTML, CSS y JS: Cada uno en su archivo para organizar mejor.
  • Valida tus formularios: Tanto en el cliente (JS) como en el servidor (si tienes backend).
